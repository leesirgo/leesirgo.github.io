Pelican 的进阶-添加主题，插件

主题查看、引用：
命令行中使用themes 命令查看主题：

pelicanconf.py文件中设置主题的选用：

  1. 选用内置主题
    THEME = "notmyidea"
  2. 选用已通过pelican-themes tool工具安装的主题
    THEME = "chunk"
  3. 通过相对路径（即相对THEME_STATIC_PATHS (['static'])指定的主题存放路径）来选用主题
    THEME = "themes/mycustomtheme"
  4. 通过绝对路径（类似：E:\leesitgoBLOG\pelican-themes\holo-pelican）来选用主题
    THEME = "~/projects/mysite/themes/mycustomtheme"

其他相关知识：
THEME_STATIC_DIR ('theme')：指定output文件夹下的主题转化后的存放路径，默认为theme子文件夹中；
THEME_STATIC_PATHS (['static'])：指定主题引用前的存放路径，即下载或clone的存放路径
CSS_FILE ('main.css')：指定CSS文件的引用路径。CSS是Cascading Style Sheets，层叠样式表，是HTLML的样式，很重要！

开始博客之路¶

metadata文件与分类¶

首先，必须先了解metadata文件，其中包含pelican将文章转化为HTML所需的各项参数，同时这些参数也与博客上文章题目、作者、分类标签等直接相关。

  1. metadata文件必须用UTF-8格式进行编码；
  2. metadata各项参数及其在博客中的用途
项目 中文名称 必要与否 用途说明
Title 标题 必要参数 博客上呈现的题目
Authors 作者 - -
Date 日期 - - -
Modified 修改日其 - -
Category 分类 - 博客中的文章分类标签，也可通过USE_FOLDER_AS_CATEGORY = True更改为用子文件名代替
Tags 标签 - -
Slug 文档的唯一标识 必要参数 pelican生成html时会直接使用这个值当html的文件名，具有唯一性，不能重复
Status 文档状态 - 当设为‘draft’时，该文章也会生成html，但不会放在博客里，可通过分享url的方式让别人看见
Summary 概述 - -

另注：
我们可以在content目录下任意建子目录来组织管理博客文章。由于我们在设置文件里指定这个值USE_FOLDER_AS_CATEGORY = True，这样这些目录名称就自动变成博文分类的目录了

常见问题和说明¶

jupyter、pelican和Github的职责分工¶

  1. jupyter文档编辑器，即博文撰写的地方；
  2. pelican是将写好的.md或者.ipynb格式文章转换为HTML格式，以便上传到网页；
  3. Github，严格来说是Github page，利用该网站提供的免费page，以个人域名的方式发布到Internet.（这句话好像很不专业。业余爱好者撰写，慎重）

引入jupyter带来的新问题¶

  1. 不同于maekdown编辑器可以在文章开头把pelican所需的metadata元数据（titile、date、slug等）直接录入，jupyter写博文需要在content文件夹下相同位置创建一个包含titile、date、slug等metadata数据的“同名.ipynb-meta”文件。
  2. 还需注意，通常这个.ipynb-meta文件通过text文本编写，需要将格式定义为UTF-8，否则博文名称出现中文后将会报错，导致该博文不能成功上传。

在pelicanconf.py配置文件中直接加载图片¶

要点：所有本地需要上传到博客的东西都在content目录下，所以在pelicanconf.py中可以用markdown语法引入图片。例如：

src="./images/profile.jpg 用的是profile.jpg相对contnet文件夹的相对路径。据说必须要用相对路径，不能用绝对路径。
